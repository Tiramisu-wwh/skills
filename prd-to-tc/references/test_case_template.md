# 测试用例模板说明

## 模板结构

基于项目内的`基础用例模板.xlsx`，测试用例表格包含以下字段：

| 字段名 | 说明 | 要求 | 示例 |
|--------|------|------|------|
| 标题* | 测试用例名称 | 必填，简洁明确描述测试目的 | "用户登录功能测试" |
| 所属分组 | 功能模块分组 | 使用"|"分隔层级 | "Web端测试用例\|首页\|我的待办" |
| 维护人 | 测试负责人 | 必须是企业成员 | "张三" |
| 前置条件 | 测试环境要求 | 描述测试前状态 | "用户已注册，系统正常运行" |
| 步骤描述 | 详细操作步骤 | 使用【1】、【2】编号 | "【1】输入用户名<br>【2】点击登录" |
| 预期结果 | 期望结果 | 与步骤描述对应 | "【1】成功进入主页<br>【2】显示用户信息" |
| 用例等级 | 优先级 | 高/中/低 | "高" |
| 用例类型 | 分类 | 功能/UI/性能/安全 | "功能测试" |

## 分组规范

### 层级结构
- 一级分组：项目模块（如：用户管理、订单处理）
- 二级分组：功能模块（如：登录注册、权限控制）
- 三级分组：具体功能（如：密码重置、账户锁定）

### 常用分组
- **功能测试**
  - 用户管理\|登录注册
  - 订单处理\|下单流程
  - 数据管理\|增删改查
- **UI测试**
  - 页面布局\|响应式设计
  - 交互体验\|操作反馈
  - 视觉效果\|颜色字体
- **性能测试**
  - 响应时间\|页面加载
  - 并发处理\|用户操作
  - 资源占用\|内存CPU
- **安全测试**
  - 身份认证\|登录验证
  - 权限控制\|访问控制
  - 数据保护\|加密传输

## 用例等级定义

### 高（High）
- 核心业务功能
- 用户高频使用功能
- 涉及资金或重要数据的功能
- 安全性要求高的功能

### 中（Medium）
- 一般业务功能
- 用户偶用功能
- 影响用户体验但非核心的功能

### 低（Low）
- 辅助功能
- 管理员功能
- 边缘场景功能

## 用例类型分类

### 功能测试
- 验证功能是否符合需求规格
- 检查业务逻辑是否正确
- 测试数据处理是否准确

### UI测试
- 验证界面显示是否正确
- 检查交互效果是否符合设计
- 测试响应式布局是否适配

### 性能测试
- 验证系统响应时间
- 检查并发处理能力
- 测试资源使用情况

### 安全测试
- 验证身份认证机制
- 检查权限控制效果
- 测试数据保护措施

### 兼容性测试
- 验证不同浏览器兼容性
- 检查不同设备适配性
- 测试不同操作系统兼容性

## 步骤描述编写规范

### 格式要求
- 使用【1】、【2】、【3】格式编号
- 每个步骤描述一个具体操作
- 步骤之间逻辑清晰，顺序合理

### 内容要求
- **明确性**：操作对象和动作要明确
- **具体性**：避免模糊描述，如"适当操作"
- **可执行性**：测试人员能够按照步骤执行
- **独立性**：每个步骤相对独立，便于问题定位

### 常用操作描述
- **输入操作**："在[字段名]输入框中输入[数据]"
- **点击操作**："点击[按钮名称]按钮"
- **选择操作**："从[下拉列表]中选择[选项]"
- **验证操作**："验证[元素]显示[状态]"

## 预期结果编写规范

### 对应关系
- 预期结果必须与步骤描述一一对应
- 使用相同的编号格式【1】、【2】、【3】

### 结果类型
- **显示结果**：页面显示的内容、状态、提示信息
- **功能结果**：功能是否执行成功，数据是否正确
- **行为结果**：系统响应、跳转、状态变化
- **性能结果**：响应时间、处理速度

### 验证要点
- **准确性**：结果是否符合业务需求
- **完整性**：是否包含所有必要信息
- **及时性**：响应是否在预期时间内
- **一致性**：多次执行结果是否一致

## 质量检查清单

在创建测试用例时，请检查以下要点：

### 必填项检查
- [ ] 标题是否填写且明确
- [ ] 维护人是否为企业成员
- [ ] 步骤描述和预期结果是否对应

### 内容质量检查
- [ ] 步骤描述是否具体可执行
- [ ] 预期结果是否明确可验证
- [ ] 分组层级是否合理
- [ ] 用例等级是否恰当

### 完整性检查
- [ ] 是否覆盖主要业务流程
- [ ] 是否包含异常情况处理
- [ ] 是否考虑边界条件
- [ ] 是否验证数据完整性

### 一致性检查
- [ ] 格式是否符合模板要求
- [ ] 术语使用是否一致
- [ ] 编号是否连续正确
- [ ] 分类是否准确
